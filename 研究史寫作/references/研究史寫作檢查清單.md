# 研究史寫作檢查清單

## 一、先問作用

- 這段研究史要服務哪個正文問題？
- 它要支撐哪個節末小結或章末結語？
- 它是在替概念定位鋪路，還是在替教化、工夫、人物、文本性質等問題鋪路？

若這三點答不出來，先不要下筆。

## 二、常見失誤

### 1. 逐家平鋪

表現：
- A 說……
- B 說……
- C 說……

問題：
- 看起來像蒐集文獻，不像整理問題。

修正：
- 改寫成「學界主要有三種處理路徑」。

### 1.5. 把文本釐定直接寫成研究史正文

表現：
- 大段交代編連、分篇、章序、校讀、異文互勘。

問題：
- 若這些內容與本節哲學論證沒有直接關聯，會擠掉真正需要整理的概念史與問題史。

修正：
- 只在文本問題會直接影響本節概念判斷時簡要交代。
- 若只是一般性文本背景，可另存為前置說明，不必寫入研究史正文。

### 2. 研究史偷渡正文主張

表現：
- 研究史裡直接斷言本文核心立場。

問題：
- 會讓正文後面的分析失去必要性。

修正：
- 改成「相關研究已揭示……但仍有待進一步釐清……」。

### 3. 研究史太短，沒有總結

表現：
- 只有幾條文獻資訊，沒有歸納。

問題：
- 讀者看不出你整理完之後的問題意識。

修正：
- 允許保留一兩句「研究史精華總結」，前提是它能被讀成對前人研究的概括。

### 4. 比較語彙喧賓奪主

表現：
- 某組分類語彙吃掉大段篇幅。

問題：
- 研究史會變成概念訓詁或作者立場預演。

修正：
- 保留必要背景。
- 若真有必要，另起一小段作比較背景。

## 三、可保留的研究史總結句

下面這類句子通常可以保留：

- 「就 X 而言，學界近年的討論大抵集中於兩個層面……」
- 「由此可見，相關研究雖已揭示……，但對於……仍未形成一致看法。」
- 「換言之，學界已逐步由……轉向……，而這也構成本文以下討論的起點。」

前提：
- 它們必須是在概括「前人研究的面向與趨勢」。
- 不能直接變成「本文已證明……」。

## 四、使用者既有成果的放法

- 若使用者自己的舊作與本題直接相關，可納入研究史。
- 放法應寫成「拙著已指出……」或「前文曾就……作初步討論」。
- 目的在於標示研究延續性，不是用來提前完成這次正文的結論。

## 五、與 Word 修訂配合

- 研究史改寫後若要進 Word，先保存一份乾淨新稿。
- compare 前先檢查：段落是否基本不動、未改 wording 是否留住。
- 若修訂來源是雲端千問回傳的純文字稿，且交付目標只是獨立節錄稿的追蹤修訂 Word，可改走 `orig.txt` + `polished.txt` -> `diff_to_tracked.py` -> `convert_tracked.py`，不必強行要求千問輸出 `<ins>/<del>`。
- compare 顆粒過粗時，再局部補 `<ins>/<del>`。
- 不要為了技術方便，把局部微調做成整段替換。
