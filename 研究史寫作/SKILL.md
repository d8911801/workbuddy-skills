# 研究史寫作

將研究史視為「替正文鋪路的學術整理」，不是作者主張的提前發表，也不是作者名單式的資料堆放。

## 何時使用

- 使用者要重寫、壓縮、重組某一節或某一章的研究史。
- 研究史需要服務正文主張、章末結語、或全書定位問題。
- 材料很多，但研究史目前像本科論文式的逐家平鋪。
- 使用者要保留自己的既有觀點，但不希望研究史與正文主張混寫。
- 研究史寫好後還要進 Word、做追蹤修訂或 compare。

## 基本流程

1. 先鎖定作用範圍。
   - 只處理使用者指定的章、節、子題。
   - 先讀最新主檔與該節上下文，不要只看單獨抽出的研究史段落。
   - 若該節有章末結語、節末小結、或全節總攝句，優先用來校準研究史的服務方向。

2. 先判斷研究史在這一節要完成什麼任務。
   - 是替某個概念定位鋪路。
   - 是替某個爭點設題。
   - 是替某個章節結論建立學術背景。
   - 是替作者既有觀點建立延續性。

3. 先按「問題群」整理，不按作者流水帳排列。
   - 文本前提或材料性質。
   - 核心概念如何被界定。
   - 與其他思想傳統或關鍵人物的比較定位。
   - 研究尚未處理或仍含混之處。
   - 若需要，可在研究史後另起一段，補「本文以下的處理方向」。
   - 若文本釐定、編連互勘、章序異同等問題與本節哲學論證沒有直接關聯，原則上不寫入研究史正文。

4. 明確分開三個層次。
   - `研究史整理`：學者說了什麼、爭點在哪裡、已有何種分類或路徑。
   - `作者主張`：本文自己的判斷、選擇、證成，原則上放到正文，不搶先塞進研究史。
   - `銜接段`：用來說明下文的分析次序、處理路徑、或本節結構。

5. 最後再檢查文體。
   - 研究史可以有總結句。
   - 但總結句必須明確是「對學界整理後的概括」，不是把本文主張偷渡進去。
   - 若一句話同時像學界總結、又像作者定論，改寫成問題揭示或比較保守的歸納。

詳細檢查點見 [references/研究史寫作檢查清單.md](references/研究史寫作檢查清單.md)。

## 硬規則

- 不要把研究史寫成作者名單接龍，除非使用者明確要求保留那種體例。
- 不要在研究史中直接完成正文的核心判斷；若需要，只提出「本文將處理的問題」。
- 文本釐定、編連、章序、校讀等材料，除非直接影響本節的哲學問題或概念判斷，否則不必寫入研究史。
- 允許保留研究史的精華總結句，但必須能被讀成「對前人研究的高度概括」。
- 若使用者有自己的舊作、專書或文章，且這些材料確實屬於當前問題史的一部分，應納入並標明其延續性；但要與本次新主張區分清楚。
- 若某組比較語彙本身就帶有後設分類性，研究史中可提示其背景，但不要讓它吃掉整段篇幅；必要時另起一小段處理。
- 若正文的實際工作包含同一出土文獻群內多篇互證、跨篇綜合或由單篇推進到整組材料的連續整理，而學界既有研究又多停留於單篇釋讀或少數篇章對讀，研究史應明確交代此一缺口與本文的處理方向；但前提是正文確已展開這項工作，不可空喊貢獻。
- 腳注配置以「一條文獻一個腳注」為優先原則；若同一句牽涉多位學者，應優先拆成數句或數分句，分別落注，不要把多位學者長串塞進同一條腳注或同一組連續標號。
- 腳注若可拆開，儘量不要在同一條腳注中堆放多條文獻。除非同一句話確實同時依據兩篇以上文獻，否則應盡量把腳注分別標在對應正文句子或分句之後，便於讀者對照。
- 腳注標號原則上放在對應正文標點之後，不限於句號，也可以是逗號、分號、冒號等句內停頓後，但要以讀者一眼能看出其所對應內容為準。
- 不可把作者與代理之間的工作語言、修稿溝通語、吸收處理說明直接寫進論文或腳注，例如「原稿所引……本文仍予吸收」「此處改置於……統觀之」一類表述，都屬內部溝通，不是讀者面向的學術文字。
- 若確實需要吸收舊稿、舊作或既有觀點，應改寫成讀者面向的學術表述：直接交代學界主張、概念路徑、思想意義或本文的分析定位，不保留修稿過程痕跡。

## 過渡段處理

研究史寫完後，若需要銜接正文，可另起一段，只做三件事：

- 回收研究史揭示出的核心問題。
- 說明本文以下的分析次序。
- 點出這些問題如何通向本節主體論證。

不要在過渡段中再重複大段研究史材料。

## 與 Word 工作流的配合

若後續要生成或回填 Word：

1. 先產出一份乾淨新稿。
2. 盡量維持原段落、句序與未改 wording 穩定。
3. 若任務錨定於既有 Word 稿件，追蹤修訂預設先走 `old/new compare`。
4. 若修訂來源是雲端千問回傳的純文字研究史稿，且交付目標是獨立節錄稿的追蹤修訂 Word，可改走 `orig.txt` + `polished.txt` -> `diff_to_tracked.py` -> `convert_tracked.py` 的本機 `md2docx` 分流，以節省 token 並提升本地 diff 粒度。
5. 只有 compare 或 `md2docx` diff 的顆粒仍太粗時，才局部回退到 `<ins>/<del>`。
6. 即使回退到 `<ins>/<del>`，也只標真實改動片段，不得整段刪除再整段新增。

涉及 Word 時，同步使用 `academic-manuscript-collaboration-workflow`。

若研究史任務同時涉及 `Word 腳注／尾註`，再加一條硬規則：

6. 不可只確認「腳注文字已出現」，還要確認「腳注編號鏈已正確顯示」。
7. 同步使用 `documents:documents`，並先讀 `tasks/footnotes_endnotes.md`。
8. 驗證至少包括三項：
   - 正文中的腳注參照號確實出現在應有位置。
   - 頁底腳注文字與編號對應正確，沒有只有文字、沒有編號的情況。
   - 在這台電腦上，預設以 Microsoft Word 匯出 PDF，再用 Poppler `pdftoppm` 轉成 PNG 做頁面檢查；確認腳注編號、換行、跨頁與版面沒有錯位，不使用 LibreOffice / `soffice` 作為渲染路徑。
9. 若是高風險交付，不以 XML 或純文字抽取視為完成，還要在 Microsoft Word 內再核一次腳注顯示。
10. `academic-manuscript-collaboration-workflow` 負責追蹤修訂與 compare；`documents:documents` 負責真實腳注／尾註、編號鏈與文件結構核對。本機版面驗證則以 Word 匯出 PDF + Poppler PNG 為準。兩者不可互相省略。

## 快速判斷

- 如果你想寫的是「我們認為……因此……」，先停一下，這多半是正文，不是研究史。
- 如果你想寫的是「學界主要集中在……但尚未充分處理……」，這通常是研究史的正確方向。
- 如果一段看起來同時像研究史與作者結論，先改成研究問題，再把真正判斷留給正文。
