## 必须遵守的硬规则

1. 先比对再写：
- 先从对方底稿抽取“台湾”段已收录人名。
- 与我方候选名单做差集。
- 对方已有条目：直接跳过，不重写。
- 仅处理“对方没有、我方有”的缺项。

2. Word 处理强制流程：
- 任何 `.docx` 任务优先使用 Documents skill 流程。
- 所有 Word 交付件必须先做乱码检查（`???`、`�`、明显中文破损）。
- 检查未通过：必须重生并复检，通过后才可交付。

3. 条目写作规则（名家条目体）：
- 生卒年：半角连字符（例：`1956-2000`）。
- 结构顺序：生平任职 → 学术职务 → 研究领域 → 学术贡献 → 主要论著。
- 固定句式：`主要从事……研究。`、`主要论著有：……`。
- `主编/合著`写在书名后括注。
- 不确定信息一律标`待确认`，不得自行补造。

4. 台湾同名校区分规则：
- `中山大学`、`交通大学`、`清华大学`如有歧义，写作`××大学（台湾）`。
- 若原文写“新竹交通大学/新竹清华大学”，可省“新竹”，保留`（台湾）`。

5. 大任务执行规则：
- 若任务非瞬时完成、循环多、文本量大或有卡顿风险，先给规模预估与分批建议，待用户确认后执行。

## 推荐执行步骤

1. 读取输入：
- 对方底稿（名家主样稿）
- 我方整合稿/人物底稿

2. 名单级去重：
- 抽取对方“台湾段”人名列表
- 抽取我方待处理人名列表
- 输出：重叠名单、缺项名单

3. 缺项写作：
- 仅对缺项人物改写
- 每批建议 3 位，便于审稿

4. 交付输出：
- 生成“补充清单”正式版
- 放入默认工作目录
- 记录“本次已补人物名单”，避免下次重复

5. 交付前QA：
- 乱码检查
- 专名核对（人名、校名、书名、学会名）
- 是否误把“国家文学博士”改写成具体校授博士（严禁推断）

- 正式交付：`台湾名家补充清单.docx`
- 审稿对比：`comparison_batch_XX_YYYY-MM-DD.docx`
