# Thesis Review Principles

This file stores the working rules distilled from repeated thesis-format reviews and teacher comments.

## Core Standard

1. Judge by visible paper appearance first.
2. Focus on formatting, not content correctness.
3. Catch problems that visibly affect readability, neatness, consistency, or formal compliance.
4. Ignore hidden technical style problems if the printed result looks acceptable.
5. Review one paper at a time.

## Review Order

1. Read the thesis file.
2. Read existing teacher comments inside the Word/WPS file if present.
3. Apply these saved principles.
4. Report only what is useful for direct correction.

## TOC Rules

1. TOC items at all levels should not begin with a two-character indent.
2. TOC entries should not use first-line indent.
3. TOC should look visually aligned and clean on paper.
4. TOC must match the real structure of the paper.
5. Missing end sections such as references or acknowledgements should be reported.
6. TOC depth should usually stop at second-level headings unless explicitly required.
7. Strange spacing, misaligned page numbers, or visually awkward dot leaders should be reported.

## Heading and Paragraph Rules

1. Same-level headings must look consistent.
2. Check font, size, weight, indentation, and spacing before and after headings.
3. Second-level headings should align with first-level headings rather than being pushed further right.
4. Normal body paragraphs usually need a two-character first-line indent.
5. Unnecessary blank lines should be removed.
6. Line spacing and paragraph spacing should be visually consistent.
7. Images, screenshots, and block quotations often break spacing and must be checked separately.
8. Same-level numbering must use one consistent pattern only.

## Front Matter Rules

1. Remove unnecessary outer brackets around Chinese and English titles.
2. Cover information lines and underlines should align.
3. Preface pages and declarations should stay visually neat and on the expected lines.
4. Chinese and English abstract placement should follow the template order.

## Abstract and Keyword Rules

1. Font size, font family, and line spacing should follow the template.
2. If the Chinese abstract changes, the English abstract should be updated too.
3. English abstract text should wrap naturally; words should not be visually stuck together.
4. Follow the template for `Key words` punctuation if the template shows a specific style.

## Punctuation and Glyph Rules

1. Chinese punctuation uses full-width forms.
2. English text and Arabic numerals use half-width forms.
3. Chinese quotation marks must use Chinese glyphs, not English straight quotes.
4. Single and double book-title marks should look visually consistent.
5. Brackets should follow the template and Chinese-writing convention.
6. Ancient-text quotations often contain stray half-width `?`, `,`, `:` and should be corrected.
7. Reference callout brackets should follow the established style; do not mix bracket shapes.

## Quotation Rules

1. Direct quotations need quotation marks unless they are block quotations.
2. Block quotations should not use quotation marks.
3. Block quotations must be introduced by the writer's own sentence first.
4. Only the quoted original text should use block-quotation formatting.
5. Block quotations should be indented as a whole, not only on the first line.
6. Leave appropriate visual space before and after block quotations.
7. Avoid sentence-by-sentence raw translation in formal academic writing when a summarized explanation is better.

## Citation and Note Rules

1. Quoted material must include source notes and page numbers when required.
2. Note markers should be placed after punctuation.
3. Citations to excavated texts and classical sources need especially careful page-location handling.
4. If a scholar's view is mentioned, the source should normally be identifiable.

## Reference Rules

1. Check references item by item, not by forcing one mechanical punctuation scheme on the whole list.
2. Always identify the item number when reporting a problem.
3. Check type markers, publication order, punctuation placement, spacing, line-end punctuation, URLs, duplicated items, and completeness.
4. DOIs are often unnecessary in this workflow unless explicitly required.
5. URLs must not be broken by inserted spaces.
6. If one classical source is cited repeatedly, prefer one authoritative edition rather than mixing editions.
7. If the body relies on an original source but the source is absent from the references, report it.

## Tables, Figures, Appendices

1. Leave one space between the table number and the table title.
2. Keep numbering rules consistent across all tables.
3. Check title font, size, and placement consistency.
4. Screenshots and appended teaching materials often disturb spacing and should be checked.
5. If screenshots are temporarily included for proof, remind the student to remove them before final submission when appropriate.

## Recurrent Teacher Preferences

These repeatedly appeared in teacher comments and should be treated as stable review checks:

1. Research-history sections should not be too thin; cited scholars, work titles, and findings should be identifiable.
2. Conclusions normally should not be packed with fresh citations.
3. Questionnaires, appendices, teaching plans, and survey materials often need separate formatting checks.
4. Traditional and simplified Chinese should not be mixed.
5. The same source name or paired source names should remain consistent throughout the paper.

## Reporting Style

1. Make the report understandable to students directly.
2. Use chapter / section / nearby sentence snippets for location.
3. Add a concrete correction example whenever possible.
4. Provide a short teacher-note version after the full report.
5. If an issue was already marked by the teacher, label it as already mentioned instead of presenting it as a fresh discovery.
