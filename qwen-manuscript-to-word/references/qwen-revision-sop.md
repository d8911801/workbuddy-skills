# 千問雲端潤稿 SOP

以下內容供每次在雲端千問進行書稿潤稿前直接貼入。

---

請依下列規則進行本次中文學術書稿潤稿，並將其視為最高優先原則：

## 一、工作模式與輸出格式

1. 採取「一節一節」或分段落的局部精修模式，確保論證層次、引文與腳注不被遺漏、改寫或重排。
2. 只輸出 **純文字 Markdown**，不要加入任何 `<ins>`、`<del>`、批註標記、修改說明或對照表。
3. 標題使用原稿既有格式；若需保留小節標題，可用 `###` 開頭。
4. 正文腳注標記一律保留為 `[^N]`，並維持在原位置。
5. 文末腳注定義一律保留為 `[^N]: 腳注內容`。
6. 獨立引文成段時，使用 `>`。
7. 段落之間空一行。
8. 不加前言、後語、提示詞回應、修訂摘要、emoji 或任何流程性文字。

## 二、內容忠實度與文獻規範

1. 保留原稿核心主張、論證方向、段落功能與文獻引用，不得擅自增刪、改換立場或提前下結論。
2. 保留楚簡、傳世文獻及學者引文中的關鍵措辭，不得因潤色而偷換概念。
3. 不遺漏、不偽造、不合併或錯置腳注。
4. 若原稿有空缺腳注，保留正式占位，例如：

```markdown
[^22]: （原稿此處腳注空缺，待補）
```

5. 腳注編號須沿用原稿，並依正文首次出現順序排列。

## 三、語言與句法

1. 潤稿目標是使文字更典雅、凝練、穩健，但不改變作者原意。
2. 現代學術論述與分析段落以「的」為主，少用「之」；古籍標題、固定術語與古典引文維持原樣。
3. 不要輸出 `<ins>` / `<del>` 標記，也不要自行模擬追蹤修訂。
4. 可修正明顯病句、錯字、學者姓名誤植與表述失衡之處，但不得擅改學術判斷。
5. 若某處涉及核心文獻判讀或學術爭議，不要自行定案；保留原文並以最小幅度整理語句。

## 四、最後檢查

輸出前確認：

1. 只有 Markdown 正文與文末腳注定義。
2. 每個 `[^N]` 都有且只有一個對應的 `[^N]: ...`。
3. 沒有遺漏引文、腳注或特殊用字。
4. 沒有 `<ins>`、`<del>`、修訂說明、對照表或額外提示語。
5. 輸出可直接交由後續本機 agent 進行 diff 與 Word 追蹤修訂生成。
