# Research Writing Rules

Use this reference for scholarly drafting, source integration, and revision.

## Scope and Source Control

- Start by naming the exact section or document unit being handled.
- Use the latest manuscript or source file named by the user.
- Do not normalize headings, terminology, or section structure silently. If inconsistent, record the issue and ask or mark it pending.
- Do not import arguments from another project unless the user asks for that comparison.

## Additive Revision

When revising an already reviewed passage:

1. Preserve the user's edited wording where possible.
2. Add missing evidence before or after the existing paragraph.
3. Add transitions rather than rebuilding the whole section.
4. Rewrite only when the existing paragraph cannot support the requested argument.
5. If rewriting is needed, explain why and preferably provide the replacement in chat first.

## Staged Stitching

When integrating material from older drafts into a newer manuscript:

- First identify reusable material by searchable opening words, not paragraph numbers.
- Distinguish material that should be fully preserved, material that should be condensed, and material that should remain only as a planning note.
- Stitch one conceptual block at a time, then inspect the local seam before moving on.
- If a newly inserted paragraph makes the next paragraph repetitive, shorten the next paragraph rather than deleting the newly approved addition.
- Do not begin global style unification, heading cleanup, citation normalization, or full-document polishing until the main body content is complete or the user explicitly asks for that pass.

## Source Evidence

- Quote the relevant source before analysis when wording matters.
- Do not use ellipses to hide key evidence.
- If a user asks to "引出所有原文", provide the full relevant span, not only the parts that support the current sentence.
- If the source has variants, identify the variant and avoid silently choosing one when it affects interpretation.
- Use structured source extraction when possible; avoid relying on memory for original texts.

## Material Layering

Keep these layers distinct:

1. primary or excavated text;
2. transmitted early text;
3. later commentary or anthology;
4. modern scholarship;
5. the user's own previous articles or notes;
6. the agent's proposed synthesis.

Do not treat these layers as equal evidence. Mark later or indirect materials as supplement, comparison, later development, or research-history context as appropriate.

## Anachronism Control

Avoid over-modernizing historical material.

- Do not directly translate ancient or premodern concepts into modern political, social, philosophical, or institutional categories without qualification.
- If using a modern concept as an analytic lens, state that it is a lens.
- Do not claim that the source material already contains the modern concept unless the evidence and historical framing support it.

## Reader-Facing Prose

Formal manuscript prose should not mention internal collaboration:

- avoid "we discussed above";
- avoid "this section should highlight";
- avoid "the next window";
- avoid "according to the user's comment";
- avoid task-management language.

Instead, write the claim directly for readers.

## Research History

When adding scholarship:

1. Name the scholar or study.
2. State its relevant contribution in one or two sentences.
3. Explain how the current draft uses, extends, qualifies, or differs from that work.
4. Put longer bibliographic or debate details in notes if they interrupt the main argument.

## Chat-Only Drafting

If the user says not to edit Word:

- provide only the passage, outline, or suggestion in the chat;
- make the text clean enough to paste;
- place uncertainties in a short note after the passage;
- do not create or modify files.
